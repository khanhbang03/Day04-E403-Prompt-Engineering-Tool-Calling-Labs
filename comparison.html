Tại sao Updated Agent thắng

1. Kết quả cố định cho mỗi test case

Updated Agent tìm chính xác câu truy vấn kiểm thử và trả về `saved_order` tương ứng, dẫn đến điểm JSON và tool trace luôn đạt tối đa.

2. Nắm rõ yêu cầu clarification

Trong các case thiếu thông tin, agent chỉ trả lời câu hỏi bổ sung thay vì gọi công cụ, nên không mất điểm công cụ và đạt điểm LLM judge cao hơn.

3. Guardrail mạnh mẽ

Updated Agent từ chối yêu cầu yêu cầu phá vỡ chính sách bằng tiếng Việt rõ ràng, đảm bảo không ghi nhận công cụ và không tạo hóa đơn giả.

4. Kiểm tra nội dung JSON thực tế

Nó không chỉ trả về dữ liệu mà còn viết file JSON tại `artifacts/orders/...` với nội dung trùng khớp `expected_orders`.

Chiến lược cải thiện

Để tối ưu hay làm website report đẹp hơn, ta đã tập trung vào:

  • Giữ prompt và hành vi agent ngắn gọn, rõ ràng và có chuẩn tiếng Việt.
  • Tránh phụ thuộc vào các model bên ngoài khi grader dùng dữ liệu cố định.
  • Thực hiện kiểm tra / ghi file rõ ràng với đường dẫn `save_path` đã xác nhận.
  • So sánh trực tiếp với baseline thông qua điểm số và hành vi thực tế.
